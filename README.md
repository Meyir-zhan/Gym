# Gym Management System - OOP Coursework

This repository contains a Python-based Gym Management System built as part of my university coursework focusing on **Object-Oriented Programming (OOP)** principles. 

## 🧠 What I Learned & Practiced
While building this project, I used AI assistance and collaboration to explore complex software architecture patterns. My primary goal was to thoroughly analyze, trace, and understand production-grade OOP design patterns, including:
- **Encapsulation:** Structuring gym members, trainers, and classes into independent modules.
- **Inheritance & Polymorphism:** Managing different user types with specialized logic.
- **Data Integrity:** Ensuring constraints (like schedule conflicts) are logically resolved.

## 🚀 Key Features
- **Member Management:** Register, track, and update gym membership statuses.
- **Class Scheduling:** Seamless booking system preventing double-booking of gym slots.
- **Trainer Assignment:** Linking distinct trainers to fitness sessions.

## 🛠️ Project Structure
- `/src`: Contains the primary application logic and OOP class declarations.
- `.gitignore`: Configured to filter out IDE-specific clutter (`.idea`, `.iml`).
